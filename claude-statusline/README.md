# Claude Code — кастомный status line

Status line для [Claude Code](https://claude.com/claude-code), который показывает
модель, папку, использование контекста и расход лимитов (квоты) — с прогресс-барами,
временем до сброса и адаптивностью под ширину терминала.

```
model: Opus 4.8  |  dir: ventoy-1.1.16  |  ctx: 40.9K (4%)  |  usage: 5h ███░░░░░ 39% (reset 2h13m) · 7d ██░░░░░░ 21% (reset 4d15h)
```

## Установка

```bash
bash install-claude-statusline.sh
```

Скрипт:

- создаёт `~/.claude/statusline-command.py` (весь код встроен в установщик);
- прописывает `statusLine` в `~/.claude/settings.json`, **не затирая** остальные
  настройки (если файл битый — делает `.bak`);
- идемпотентный: можно запускать повторно;
- учитывает переменную `CLAUDE_CONFIG_DIR`, если она задана.

После установки **перезапусти Claude Code** (или начни новую сессию).

Установка глобальная — бар работает во всех проектах.

## Что показывает

| Блок     | Данные                                              | Источник (JSON status line)                |
|----------|-----------------------------------------------------|--------------------------------------------|
| `model:` | Название модели                                     | `model.display_name`                       |
| `dir:`   | Имя текущей папки (обрезается до 24 симв. с `…`)     | `cwd`                                       |
| `ctx:`   | Токены в контексте + % окна                          | `context_window.current_usage` / `used_percentage` |
| `usage:` | Расход 5-часового и недельного лимита + время сброса | `rate_limits.five_hour` / `seven_day`      |

Цвет процентов: зелёный `<50%`, жёлтый `<80%`, красный `>=80%`.

## Адаптивность

Строка подстраивается под ширину терминала. Сначала убираются прогресс-бары у
`usage`, и только если и без баров не помещается — отбрасываются целые блоки по
приоритету (что правее — исчезает первым):

```
usage 5h  >  dir  >  usage 7d  >  model  >  ctx
```

Самый приоритетный блок (`usage 5h`) показывается всегда.

## Настройка

В начале `~/.claude/statusline-command.py`:

- `MAX_DIR` — максимальная длина имени папки до обрезки (по умолчанию `24`);
- ANSI-константы (`CYAN`, `YELLOW`, …) — цвета блоков;
- `bar(pct, width=8)` — ширина прогресс-бара;
- приоритеты/порядок блоков — в вызовах `add_block(...)` (аргументы `prio`, `order`).

## Известное ограничение

Claude Code вызывает скрипт только при обновлении интерфейса (новое сообщение,
ответ, активность), не по таймеру. Поэтому в простое таймеры `(reset …)` и проценты
не «тикают» — они актуализируются при следующем действии. Это поведение самого
Claude Code, а не скрипта.

## Удаление

Убери ключ `statusLine` из `~/.claude/settings.json` и удали
`~/.claude/statusline-command.py`.
